Root directory

folder app

The core code of your application. Autoloaded via PSR4.

folder bootstrap

Files that bootstrap the framework and configure autoloading.

folder bootstrap/compile

Contains the compilation of various elements of the applications (code, config, routes, ...).

folder config

Your application's configuration files.

folder migrations

Contains your database migrations

folder public

The entry point for all requests entering your application.

folder resources

Your views as well as your raw, uncompiled assets such as LESS, SASS, or JavaScript.

folder routes

All of the route definitions for your application.

folder storages

All logs & caches & compiled views.

folder tests

Your automated tests.

App directory

folder Core

Contains your application core code. Should contains business logic, shared with all kernels.

folder Core/Constants

This directory contains all constants you may should use in your application.

folder Core/Facades

This directory contains all facades you may should use in your application.

folder Core/Models

The Models directory is a proposed location of your Models.

folder Core/Providers

The Providers directory contains all of the service providers for your application. Service providers bootstrap your application by binding services in the service container (Phalcon\Di) without instantiating.

folder Core/Support

The Support directory is a proposed location of your tools.

folder Kernels

Contains the code the application kernels (Kernel settings, Controllers, ... ).

folder Kernels/Cli Console application

The Tasks directory contains all of the custom cli commands for your application. This directory also houses your console kernel.

folder Kernels/Http Http fullstack application

The Http directory contains your controllers, middleware, modules, and your http kernel.

folder Kernels/Micro Micro application

The Micro directory contains your controllers, middleware, and your micro kernel.
