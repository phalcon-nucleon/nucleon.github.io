PHP - Opcache

To give your application a huge speed boost, you must use a PHP Bytecode Cache.

Prefer Zend Opcache, is only opcode cache. Built for this, works for this.

Opcache documentation

Configuration example :

[opcache]
; Determines if Zend OPCache is enabled
opcache.enable=1
; The OPcache shared memory storage size.
opcache.memory_consumption=64 ; Or more for server multi-site
; The amount of memory for interned strings in Mbytes.
opcache.interned_strings_buffer=16
; The maximum number of keys (scripts) in the OPcache hash table.
opcache.max_accelerated_files=20000
; Enabled a fast shutdown sequence
opcache.fast_shutdown=1
; Reduce the size of the optimized code.
opcache.save_comments=0
; Disable check for updated scripts.
opcache.validate_timestamps=0
opcache.validate_timestamps

If it's disabled, your PHP application will be completely cached, so you'll get the maximum performance.
But you must reset the OPcache manually or restart the webserver for changes to the filesystem to take effect.

You can very well plan in your deployment routine to restart php-fpm or other instances.

Nucleon does not use annotations, so saving comments in opcache is useless.

In general, annotations are bad practice.
In Java, C#, ..., annonations aren't in comment blocks, and they are compile and used by the language.
In PHP, annonations are in comment blocks, and a comment is not code, it's a comment.
